fix(research): add ResearchRequestFailed for retryable OpenAI status='failed'

OpenAI's Responses API can return status='failed' intermittently server-side.
Previously this raised a bare RuntimeError which was not classified as transient,
causing permanent pipeline failure on a recoverable condition.

Changes:
- openai_deep.py: add ResearchRequestFailed(RuntimeError) exception; extract and
  log error.message from the API response before raising, so diagnostic detail is
  preserved rather than discarded
- multi_runner.py: add 'ResearchRequestFailed' to _is_research_transient() transient_names
  set using the existing class-name matching pattern (no hard import dependency)
- Tests: classifier test, MockFailedRequestProvider test double, end-to-end retry
  integration test, and error-detail extraction test

Plain RuntimeError remains non-transient — no change to existing failure behaviour.
